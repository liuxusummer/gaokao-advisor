## 1. 项目背景与目标

### 1.1 项目背景

2026 年全国高考报名人数达 1290 万，其中县域考生占比 57.27%（超 700 万人）。志愿填报是考生和家长最焦虑的环节，面临三重困境：

- **信息差**：近三成考生对填报规则一无所知，19.4% 担忧滑档，35.4% 害怕选错专业
- **资源差**：全国真正获得专业指导的家庭不足 5%，仅 23.5% 高中开设系统职业规划课程
- **指导差**：付费咨询价格高昂（3,000-18,999 元）且央视报道服务注水严重；市面 AI 志愿工具政策匹配错误率高达 31.2%

### 1.2 项目目标

打造一个**普惠、可解释、高准确率**的纯前端 Web 应用，将专业级志愿规划能力以零成本方式提供给全国考生，尤其帮助县域和农村地区考生打破信息壁垒。

### 1.3 核心目标（SMART）

| 目标维度 | 具体指标 |
|---------|---------|
| **准确性** | 政策匹配错误率 < 5%（行业平均 31.2%），每条推荐附数据来源 |
| **普惠性** | 完全免费，纯前端运行，无需注册即可使用核心功能 |
| **可解释性** | 每条志愿推荐提供录取概率、位次对比、推荐理由 |
| **覆盖范围** | 支持新高考 29 省份，覆盖"专业+院校"和"院校专业组"两种模式 |
| **响应速度** | 核心推荐生成 < 3 秒，页面首屏加载 < 2 秒 |
| **离线可用** | 核心推荐功能在无网络环境下可用（基于内置数据库） |

| 实际付费家庭占比 | < 5% | 公开数据 |

### 2.2 竞品分析

| 产品 | 定位 | 商业模式 | AI 能力 | 核心问题 |
|------|------|---------|---------|---------|

### 2.3 行业核心痛点

1. **AI 工具准确率不足**：市面主流 AI 志愿工具基础政策匹配错误率高达 31.2%（来源：《2026 教育 AI 产品实测白皮书》），典型错误包括推荐已停招专业、选科要求不符、异地办学点误判为本部
2. **数据非实时同步**：除教育部"阳光志愿"外，商业产品数据无法与各省考试院实时同步，是错误率的根因
3. **服务注水严重**：央视/法治日报报道，"高考志愿规划师"无资质门槛，三版方案两版一致，直接套用模板
4. **能力断层**：免费 AI 拉平了信息差，但县域学生不会用、不敢用、用不好 AI 工具，反而在信息过载中更迷茫

### 2.4 差异化机会

本产品的差异化定位：**官方数据 + 强 AI 解释性 + 政策校验 + 引用溯源 + 普惠免费**

- 现有产品解决"推荐什么"，未解决"为什么推荐、如何判断对错、如何与官方核对"
- 谁能把错误率降到个位数，谁就能建立信任壁垒
- 700 万县域考生 + 57.27% 占比 + 95% 家庭未付费，是未被充分服务的蓝海

## 3. 用户画像与需求分析

### 3.1 目标用户

#### 用户角色 1：高考考生（主要用户）

- **特征**：17-19 岁，面对 112 个志愿填报名额和 883 种专业不知从何下手
- **场景**：高考出分后至志愿填报截止前的黄金 3-7 天内
- **核心需求**：
  - 输入成绩位次，快速获得个性化志愿方案
  - 了解专业是否适合自己
  - 确认志愿表无滑档/退档风险
- **痛点**：对填报规则一无所知，担忧滑档和选错专业

#### 用户角色 2：考生家长（决策参与者）

- **特征**：缺乏专业知识，担心被高价咨询"割韭菜"
- **场景**：协助孩子填报志愿，需要理解推荐理由
- **核心需求**：
  - 看懂推荐方案的合理性
  - 确认志愿表安全可靠
  - 获取权威政策解读
- **痛点**：信息不对称，难以判断 AI 推荐的可靠性

#### 用户角色 3：县域/农村考生（重点普惠对象）

- **特征**：占考生总数 57.27%，信息资源严重匮乏
- **场景**：学校老师说不清专业区别，家长靠"学医好、计算机热门"朴素判断
- **核心需求**：
  - 零成本获得专业级指导
  - 操作简单，无需专业背景
  - 离线可用（网络条件差）
- **痛点**：三重困境（信息差、资源差、指导差）

#### 用户角色 4：中学教师/志愿规划师（专业用户）

- **特征**：提供志愿填报指导的专业人员
- **场景**：批量指导学生填报志愿
- **核心需求**：
  - 快速生成多份个性化方案
  - 数据准确可核查
  - 支持导出和对比
- **痛点**：手工比对数据耗时数周

## 4. 产品定位与价值主张

### 4.1 产品定位

**普惠型 AI 高考志愿填报助手** — 纯前端 Web 应用，核心交互为 AI 对话 + 可视化数据面板。

### 4.2 价值主张

> **让每一分都不浪费，让每一个家庭都不焦虑。**

| 价值维度 | 具体内容 |
|---------|---------|
| **社会价值** | 将专业级志愿规划能力以零成本方式普惠给全国考生，尤其帮助县域和农村地区 57.27% 考生群体打破信息壁垒 |
| **效率价值** | 传统志愿填报需翻阅大量资料、反复比对数据，耗时数周；AI 助手将这一过程压缩到几分钟 |
| **信任价值** | 每条推荐附数据来源和引用链接，用户可一键核查，直击行业 31.2% 错误率痛点 |
| **安全价值** | 滑档/退档风险预警系统，有效降低滑档风险（部分省份滑档率高达 5%-8%） |

### 4.3 核心差异化

1. **可解释性**：每条推荐提供录取概率、位次对比、三年数据趋势、推荐理由
2. **引用溯源**：AI 联网搜索结果附权威来源链接，优先采用教育考试院、阳光高考等官方来源
3. **政策校验**：内置政策规则引擎，自动检测选科不符、身体条件受限、单科不足等退档风险
4. **离线可用**：核心推荐基于内置数据库，无网络环境下完整可用
5. **LLM 可选**：用户可选接入云端 LLM API 增强对话能力，不接入不影响核心功能

### 5.2 功能需求详细说明

#### FR-01：用户画像录入

**优先级**：P0

**功能描述**：用户录入高考成绩、位次、选科、偏好等信息，作为推荐基础。

**输入字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 省份 | 下拉选择 | 是 | 29 个新高考省份 |
| 科类 | 下拉选择 | 是 | 物理类/历史类（3+3 模式为"综合"） |
| 高考总分 | 数字 | 是 | 0-900（海南满分 900，上海满分 660，其余 750） |
| 全省位次 | 数字 | 是 | 从一分一段表查询 |
| 选考科目 | 多选 | 是 | 3 门（3+3）或 3 门（3+1+2，含首选物理/历史） |
| 地域偏好 | 多选 | 否 | 北上广深/新一线/省会/其他/不限 |
| 院校类型偏好 | 多选 | 否 | 985/211/双一流/省重点/普通本科/不限 |
| 学费上限 | 数字 | 否 | 元/年，默认不限 |
| 专业方向偏好 | 多选 | 否 | 13 个学科门类 |
| 体检结论 | 下拉 | 否 | 正常/色弱/色盲/视力异常/身高受限/其他 |
| 单科成绩 | 数字组 | 否 | 语文/数学/英语/物理/化学/生物/政治/历史/地理 |

**业务规则**：
- 省份选定后，自动加载该省份的志愿模式（专业+院校 / 院校专业组）和志愿数量
- 位次必须与总分匹配（通过一分一段表校验），不匹配时提示用户核查
- 选科确定后，自动过滤不符合选科要求的专业
- 体检结论影响退档风险检测

**验收标准**：
- 表单字段完整，必填项校验生效
- 省份切换后志愿模式自动切换
- 数据本地持久化，刷新不丢失

#### FR-02：等效位次换算

**优先级**：P0

**功能描述**：将考生当年位次换算为往年等效位次，用于跨年对标历史录取数据。

**输入**：当年位次、省份、科类

**输出**：近 3-5 年等效位次和等效分

**算法**：
1. 从内置一分一段表中查找当年位次对应的总分（验证用）
2. 查找往年一分一段表中相同累计位次对应的分数（等效分）
3. 输出近 3-5 年的等效分和等效位次

**展示**：
- 表格形式展示近 5 年等效分变化趋势
- 折线图展示位次稳定性

**验收标准**：
- 换算结果与各省考试院发布的一分一段表一致
- 支持手动修正位次（用户从考试院官网查询后输入）

#### FR-03：冲稳保志愿推荐

**优先级**：P0

**功能描述**：基于位次法和冲稳保梯度策略，生成个性化志愿推荐方案。

**输入**：用户画像（成绩/位次/选科/偏好）、等效位次

**输出**：冲稳保三档志愿列表，每个志愿包含：
- 院校名称、专业名称、专业组代码
- 院校类型（985/211/双一流）
- 近 3 年录取最低分/位次/平均分
- 录取概率（百分比）
- 推荐理由（位次对比、梯度归属）
- 数据来源链接

**梯度划分规则**（基于等效位次）：

| 梯度 | 位次浮动范围 | 录取概率 | 志愿数量占比 |
|------|------------|---------|------------|
| 冲 | 等效位次上浮 5%-15%（位次数字更小） | 20%-40% | 20%-30% |
| 稳 | 等效位次 ±5% | 60%-80% | 40%-60% |
| 保 | 等效位次下浮 10%-20%（位次数字更大） | 90%+ | 20%-30% |

**不同省份志愿数量分配**：

| 志愿总数 | 冲 | 稳 | 保 | 适用省份示例 |
|---------|----|----|----|------------|
| 112 | 28-34 | 45-56 | 22-34 | 辽宁 |
| 96 | 19-29 | 38-58 | 19-29 | 河北、山东、重庆 |
| 80 | 16-24 | 32-48 | 16-24 | 浙江 |
| 45 | 9-13 | 18-27 | 5-13 | 湖北、湖南、广东 |
| 40 | 8-12 | 16-24 | 4-12 | 江苏、福建 |
| 30 | 6-9 | 12-18 | 3-9 | 北京 |
| 24 | 5-7 | 10-14 | 3-7 | 上海 |

**业务规则**：
- 自动过滤不符合选科要求的院校专业
- 自动过滤体检受限专业（如色弱不推荐医学类）
- 自动过滤学费超限专业
- 支持用户调整梯度比例（滑块控制冲稳保占比）
- 支持用户手动锁定某些志愿后重新推荐其余志愿
- 每条推荐附数据来源（阳光高考/省考试院）和更新日期

**验收标准**：
- 推荐结果符合冲稳保梯度规则
- 选科/体检/学费过滤生效
- 每条推荐可查看详细数据来源
- 推荐生成时间 < 3 秒

#### FR-04：专业兴趣测评

**优先级**：P1

**功能描述**：通过霍兰德 RIASEC 测评和学科兴趣测评，推荐匹配的专业方向。

**子功能 4.1：霍兰德 RIASEC 测评**

- **题量**：60 题（每维度 10 题，5 级李克特量表）
- **时长**：约 10 分钟
- **输出**：3 字母霍兰德代码（如 SAE、IRC）+ 六维度雷达图
- **映射**：霍兰德代码 → 13 学科门类 → 92 专业类 → 883 专业

**子功能 4.2：学科兴趣测评**

- **题量**：15 题（12 题学科兴趣 + 3 题行为倾向）
- **时长**：约 3 分钟
- **输出**：前 3 高分学科 + 推荐专业大类

**子功能 4.3：测评结果整合**

- 交叉验证霍兰德与学科兴趣结果
- 一致时标注"高置信度"，不一致时提示深入测评
- 测评结果可应用于推荐排序（专业兴趣维度加权）

**RIASEC 与学科门类映射**：

| 霍兰德代码 | 对应学科门类 |
|-----------|------------|
| R 现实型 | 工学、农学 |
| I 研究型 | 理学、医学（基础）、工学（计算机/电子） |

## 9. 核心推荐算法

### 9.1 算法流程

```
输入：用户画像（成绩/位次/选科/偏好/体检）
  │
  ▼
[1] 等效位次换算
  │  从一分一段表查近 3-5 年等效位次
  ▼
[2] 候选池筛选
  │  按选科/体检/学费/地域/院校类型过滤
  ▼
[3] 录取概率计算
  │  基于近 3 年加权平均位次估算概率
  ▼
[4] 冲稳保梯度划分
  │  按位次浮动范围分为冲/稳/保三档
  ▼
[5] 多维度排序
  │  叠加专业兴趣/地域/院校类型等维度
  ▼
[6] 志愿数量分配
  │  按省份志愿数量分配冲稳保数量
  ▼
输出：冲稳保三档志愿列表（含概率/理由/来源）
```

### 9.2 等效位次换算

**输入**：当年位次 N、省份、科类

**算法**：
1. 从一分一段表查找当年位次 N 对应的总分（验证）
2. 对近 3-5 年每年的一分一段表：
   - 查找累计位次 = N 对应的分数，即为该年等效分
3. 输出近 3-5 年等效分列表

**示例**：
- 2026 年位次 18000 名
- 2025 年位次 18000 名对应 580 分（2025 等效分）
- 2024 年位次 18000 名对应 575 分（2024 等效分）
- 2023 年位次 18000 名对应 582 分（2023 等效分）

### 9.3 录取概率计算

**输入**：考生等效位次、目标院校专业近 3 年录取位次

**算法**：
1. 计算目标院校专业近 3 年录取位次的加权平均（近期权重高）：
   - 基准位次 = 去年位次 × 0.5 + 前年位次 × 0.3 + 大前年位次 × 0.2
2. 计算偏差率：`偏差率 = (考生位次 − 基准位次) / 基准位次`
3. 按偏差区间映射概率：

| 偏差率 | 录取概率 | 梯度归属 |
|--------|---------|---------|
| < -15%（考生位次远优于院校） | 90%+ | 保 |
| -15% ~ -5% | 80%-90% | 保 |
| -5% ~ +5% | 60%-80% | 稳 |
| +5% ~ +15% | 20%-40% | 冲 |
| > +15%（考生位次劣于院校） | < 20% | 不推荐 |

**大小年处理**：
- 计算近 3 年位次标准差
- 标准差 > 15% 时标注"大小年风险"，降低概率 10%
- 标准差 > 25% 时标注"高风险波动"，降低概率 20%

### 9.4 冲稳保梯度划分

**基于等效位次的百分比浮动**：

| 梯度 | 位次浮动范围 | 录取概率 | 说明 |
|------|------------|---------|------|
| 冲 | 等效位次上浮 5%-15% | 20%-40% | 位次数字更小，排名更靠前 |
| 稳 | 等效位次 ±5% | 60%-80% | 核心主力 |
| 保 | 等效位次下浮 10%-20% | 90%+ | 位次数字更大，兜底防滑档 |

### 9.5 多维度排序

在位次法圈定候选池后，叠加以下维度排序：

| 维度 | 权重（默认） | 说明 |
|------|------------|------|
| 录取概率 | 30% | 位次匹配度 |
| 院校层次 | 25% | 985/211/双一流/省重点 |
| 专业兴趣 | 20% | 测评结果匹配度 |
| 地域偏好 | 15% | 城市层级匹配 |
| 学费 | 5% | 经济承受力 |
| 就业前景 | 5% | 行业发展趋势 |

**权重可调**：用户可通过滑块调整各维度权重，实时更新排序结果。

**分类型权重建议**：
- 高分段考生：院校层次权重上调至 40%+
- 中分段考生：专业权重上调至 30%
- 低分段考生：录取概率和地域权重上调

## 10. 风险预警规则

### 10.1 滑档风险检测

**检测逻辑**：

```
输入：考生位次、志愿表（按顺序的院校专业及往年录取位次）
  │
  ▼
[1] 冲稳保比例检测
  │  统计冲/稳/保数量，判断比例是否合理
  ▼
[2] 保底院校位次差检测
  │  检查最后一个志愿的位次是否足够低
  ▼
[3] 志愿间梯度检测
  │  检查相邻志愿位次差是否合理（无断层）
  ▼
[4] 数据年限检测
  │  检查推荐依据的数据年限
  ▼
[5] 大小年波动检测
  │  计算目标院校近 3 年位次标准差
  ▼
输出：滑档风险等级（高/中/低）+ 具体风险项
```

**风险等级判定**：
- **高风险**：满足 2 项及以上高风险条件，或保底院校位次差 < 5%
- **中风险**：满足 1 项高风险或 2 项中风险
- **低风险**：无高风险，中风险 ≤ 1 项

### 10.2 退档风险检测

**检测维度**：

1. **服从调剂检测**（院校专业组模式）
   - 未勾选服从调剂 → 高风险
   - 专业组内仅 1 个专业且不服从调剂 → 高风险

2. **身体条件检测**
   - 色盲填报医学、化学、材料等限报专业 → 高风险
   - 色弱填报医学类 → 高风险
   - 视力不合格填报航海/军警 → 高风险
   - 身高受限填报对应专业 → 高风险

3. **单科成绩检测**
   - 单科成绩低于章程要求 → 高风险
   - 单科成绩接近要求线（差距 ≤ 3 分）→ 中风险

4. **选科匹配检测**
   - 选科不含必选科目（如未选物理报工科）→ 高风险

5. **政审/体检/面试检测**
   - 填报军警公安类但未参加政审/体检 → 高风险

6. **专业级差检测**
   - 院校有级差且排序未考虑级差 → 高风险

### 10.3 体检受限专业规则

依据《普通高等学校招生体检工作指导意见》（教学〔2003〕3 号）：

**色觉受限专业**：
- **色弱**：医学类、化学类、材料类、环境类、制药类、生物科学类、心理学类、地质学类、农学类、医学技术类、艺术设计（部分）限报
- **色盲**：除色弱限报专业外，美术学、绘画、艺术设计、摄影、动画等专业限报
- **可报专业**：哲学、法律、社会学、社会工作、文秘、数学、历史、教育学、英语、新闻、汉语言等

**视力受限专业**：
- 裸眼视力任何一眼低于 5.0：飞行技术、航海技术、消防工程、刑事科学技术、侦察
- 裸眼视力任何一眼低于 4.8：轮机工程、运动训练、民族传统体育

**身高受限专业**：
- 男性身高低于 170cm/女性低于 160cm：部分航海、公安、军事、旅游管理专业

## 11. 专业兴趣测评设计

### 11.1 霍兰德 RIASEC 测评

**六类型定义**：

| 代码 | 中文名 | 核心特征 | 典型活动 |
|------|--------|---------|---------|
| R | 现实型 | 喜欢与物体、机器、工具打交道；动手能力强 | 机械操作、工程、户外作业 |
| I | 研究型 | 喜欢观察、分析、解决问题；逻辑思维强 | 科研、编程、数据分析 |
| A | 艺术型 | 追求创造力、自由表达；想象力丰富 | 设计、写作、音乐、表演 |
| S | 社会型 | 乐于助人、重视人际关系 | 教学、咨询、社工、护理 |
| E | 企业型 | 喜欢领导、影响、说服他人 | 销售、管理、创业、公关 |
| C | 常规型 | 喜欢组织、处理数据；偏好规则明确 | 会计、行政、统计、文秘 |

**测评设计**：
- **题量**：60 题（每维度 10 题）
- **题型**：5 级李克特量表（1=完全不喜欢，5=非常喜欢）
- **时长**：约 10 分钟
- **计分**：按维度求和，取前 3 高分维度组成霍兰德代码（如 SAE）
- **输出**：六维度雷达图 + 3 字母霍兰德代码 + 推荐专业大类

### 11.2 学科兴趣测评

**测评设计**：
- **题量**：15 题（12 题学科兴趣 + 3 题行为倾向）
- **题型**：5 级李克特量表
- **时长**：约 3 分钟
- **计分**：按学科得分排序，取前 3 高分学科
- **输出**：前 3 高分学科 + 推荐专业大类

**学科—专业映射**：

| 学科兴趣 | 推荐专业大类 |

| 生物/生命 | 基础医学类、临床医学类、生物科学类 |
| 语文/写作 | 中国语言文学类、新闻传播学类 |
| 英语/语言 | 外国语言文学类、翻译 |
| 政治/辩论 | 法学类、政治学类 |
| 历史/文化 | 历史学类、考古学、文物与博物馆学 |
| 地理/环境 | 地理科学类、环境科学与工程类、城乡规划 |

## 12. AI 对话能力设计

### 12.1 LLM 集成方案

**接入方式**：用户自带 API Key，前端直接调用云端 LLM API。

**支持的 LLM 提供商**（初始支持）：
- 智谱 GLM
- 阿里通义千问
- OpenAI GPT
- 百度文心一言
- 其他兼容 OpenAI API 格式的服务

**配置项**：
- API Base URL
- API Key
- 模型名称
- 温度参数（默认 0.3，保证回答稳定性）

### 12.2 对话场景

| 场景 | 说明 | 知识来源 |
|------|------|---------|
| 政策规则问答 | 解答新高考政策、志愿填报规则 | 内置政策知识库 + LLM |
| 推荐解释 | 针对某条推荐追问理由 | 推荐引擎数据 + LLM |
| 院校专业咨询 | 询问院校/专业详情 | 内置数据库 + LLM |
| 联网搜索增强 | 获取最新政策/招生计划 | LLM 联网搜索 + 引用 |

### 12.3 防幻觉机制

- LLM 回答基于内置政策知识库（System Prompt 注入）
- 联网搜索结果必须附引用来源链接
- 关键数据（分数线/位次/政策）以内置数据库为准，LLM 仅做解释
- LLM 不得自行承诺录取结果
- 回答末尾标注"以上信息仅供参考，请以官方发布为准"

### 12.4 隐私与安全

- API Key 加密存储于本地（IndexedDB）
- API Key 不上传任何服务器
- 对话历史本地存储
- 不收集用户对话内容用于训练

> 2023 年统计：35% 的退档考生总分过线，却因单科差 2 分、体检色弱、不知道专业级差等原因被退档。

### 16.6 术语表

| 术语 | 定义 |
|------|------|
| 位次 | 考生在全省同科类考生中的累计排名 |
| 等效位次 | 跨年位次对比时，不同年份相同位次对应的分数 |
| 一分一段表 | 省考试院发布的分数-位次对照表 |
| 冲稳保 | 志愿梯度策略：冲（冲刺更好院校）、稳（核心主力）、保（兜底防滑档） |
| 滑档 | 所有志愿分数都不够，档案未投出 |
| 退档 | 档案已投出但被院校退回 |
| 院校专业组 | 新高考志愿单位，1 个专业组含多个专业，组内可调剂 |
| 专业+院校 | 新高考志愿单位，1 个志愿 = 1 个专业 + 1 个院校，无调剂 |
| 大小年 | 院校录取位次年度大幅波动的现象 |
| 线差法 | 用考生分数与批次线的差值对标院校的方法 |
| 位次法 | 用全省排名对标院校历史录取数据的方法 |
| 霍兰德代码 | RIASEC 六类型中得分前 3 的组合，如 SAE |
| 特控线 | 特殊类型录取控制分数线，用于强基计划、军校等 |
