## 1. 设计原则与目标

### 1.1 UX 设计原则（P0 级要求）

| 原则 | 说明 | 落地体现 |
|------|------|---------|
| **极简优先** | 核心路径 3 步完成，隐藏高级选项 | 4 步向导：省份→成绩→偏好→结果 |
| **不制造焦虑** | 风险预警客观呈现，提供解决方案 | 实时内联标记 + 一键修复 |
| **可解释可溯源** | 每条推荐附理由和数据来源 | 内联引用标注 + 来源列表 |
| **移动优先** | 县域用户 57.27% 为手机访问 | 底部 Tab + 单列卡片流 |
| **渐进增强** | 核心功能离线可用，AI 为增强 | LLM 未接入时优雅降级 |
| **信任感** | 权威数据 + 透明算法 + 免责声明 | 引用溯源 + 算法说明 |

### 1.2 设计目标（SMART）

| 目标维度 | 具体指标 |
|---------|---------|
| **任务完成率** | 核心路径（向导→推荐→保存）完成率 > 85% |
| **操作效率** | 从进入到生成推荐 < 90 秒 |
| **首次理解率** | 推荐结果首次理解率 > 80%（用户调研） |
| **移动端可用性** | 移动端任务完成率与桌面端差距 < 10% |
| **信任度** | 90% 用户注意到数据来源标注 |
| **无障碍** | 满足 WCAG 2.1 AA 级标准 |

## 2. 用户旅程地图

### 2.1 核心用户旅程（考生/家长）

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  认知    │ →  │  入门    │ →  │  核心使用 │ →  │  决策    │ →  │  分享    │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
   │              │              │              │              │
   ▼              ▼              ▼              ▼              ▼
听说产品       首页引导        填报向导        对比方案       导出志愿表
搜索找到       示例体验        生成推荐        风险检查       分享给家人
              兴趣测评        调整偏好        AI 咨询       提交确认
```

### 2.2 情绪曲线分析

| 阶段 | 用户情绪 | 痛点 | 设计对策 |
|------|---------|------|---------|
| 认知 | 好奇+怀疑 | "又一个志愿工具？" | 首页突出差异化：免费+溯源+准确率 |
| 入门 | 期待+迷茫 | "从哪里开始？" | 极简向导，1 个主 CTA |
| 核心使用 | 专注+焦虑 | "我的分能上什么？" | 3 秒生成推荐，结果即时可见 |
| 决策 | 纠结+担忧 | "会不会滑档？" | 风险预警 + AI 咨询 + 数据溯源 |
| 分享 | 释然+不确定 | "家人怎么看？" | 导出 PDF/图片，扫码同步 |

```
│ 🎓 智填志愿   首页  推荐  志愿表  测评  数据  AI  ⚙ │
└──────────────────────────────────────────────────────┘

移动端（< 768px）：
┌──────────────────────┐
│ 🎓 智填志愿       ⚙  │
├──────────────────────┤
│                      │
│      内容区域        │
│                      │
├──────────────────────┤
│ 🏠  🎯  📋  📊  💬  │
│ 首页 推荐 志愿 数据 AI│
└──────────────────────┘
```

### 4.2 导航项定义

| 导航项 | 图标 | 桌面标签 | 移动标签 | 路由 | 说明 |
|--------|------|---------|---------|------|------|
| 首页 | 🏠 | 首页 | 首页 | `/` | 向导入口 + 快捷功能 |
| 推荐 | 🎯 | 推荐 | 推荐 | `/recommend` | 推荐结果（无数据时引导填写） |
| 志愿表 | 📋 | 志愿表 | 志愿表 | `/volunteer-list` | 已选志愿管理（带风险标记） |
| 测评 | 🧭 | 测评 | — | `/assessment` | 仅桌面端导航，移动端从首页进入 |
| 数据 | 📊 | 数据 | 数据 | `/data` | 院校/专业/分数线查询 |
| AI | 💬 | AI | AI | `/chat` | AI 问答（未接入时显示引导） |
| 设置 | ⚙️ | ⚙ | ⚙ | `/settings` | 右上角图标 |

### 4.3 移动端底部 Tab 策略

- **5 个 Tab 上限**：首页 / 推荐 / 志愿表 / 数据 / AI
- **测评入口**：移动端从首页卡片进入，不占 Tab 位
- **设置入口**：移动端右上角齿轮图标
- **Tab 徽标**：志愿表有风险时显示红色数字徽标

## 6. 核心流程交互设计

### 6.1 核心路径：4 步向导（已确认极简向导优先）

```
Step 1: 选择省份      Step 2: 输入成绩      Step 3: 选择偏好      Step 4: 查看结果
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 你在哪里高考？│    │ 你的成绩是？  │    │ 你的偏好是？  │    │ 推荐结果      │
│              │    │              │    │              │    │              │
│ [省份选择器]  │ →  │ [分数输入]    │ →  │ [兴趣(可选)]  │ →  │ [冲稳保标签]  │
│ [科类选择]    │    │ [位次输入]    │    │ [地域偏好]    │    │ [志愿卡片]    │
│ [选科信息]    │    │ [选科确认]    │    │ [专业偏好]    │    │ [筛选排序]    │
│              │    │              │    │              │    │              │
│ [下一步]      │    │ [下一步]      │    │ [生成推荐]    │    │ [加入志愿表]  │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

### 6.2 向导交互细节

#### Step 1：选择省份

- **标题**："你在哪里高考？"
- **必填**：省份/直辖市
- **选填**：科类（新高考自动判断，旧高考选择文理）
- **选填**：选科（新高考省份显示，3+3 或 3+1+2）
- **默认值**：根据 IP 定位省份（可修改）
- **下一步条件**：省份已选
- **进度指示**：顶部进度条 `1/4`

#### Step 2：输入成绩

- **标题**："你的成绩是？"
- **必填**：高考分数（数字输入，0-750）
- **选填**：位次（如已知，可提高推荐精度）
- **自动填充**：分数输入后，若内置该省当年一分一段表，自动估算位次
- **校验**：分数范围 0-750（部分省份 0-900），超出提示
- **下一步条件**：分数已填且合法
- **进度指示**：`2/4`

#### Step 3：选择偏好（可跳过）

- **标题**："你的偏好是？（可跳过）"
- **可选模块**：
  - 兴趣方向：从兴趣测评结果导入，或手动选择 3-5 个兴趣标签
  - 地域偏好：多选期望省份/城市，或"不限"
  - 专业偏好：多选期望专业类，或"不限"
  - 院校偏好：985/211/双一流/省重点/不限
  - 风险偏好：保守/均衡/激进（影响冲稳保比例）
- **跳过逻辑**：点击"跳过，使用默认偏好"直接进入下一步
- **进度指示**：`3/4`

#### Step 4：查看结果

- **标题**："为你推荐的志愿"
- **加载态**：骨架屏 + "正在为你匹配最佳志愿..."（< 3 秒）
- **结果展示**：见第 7 节
- **进度指示**：`4/4 完成`

### 6.3 向导通用交互

- **返回**：每步左上角"← 上一步"，不丢失已填数据
- **持久化**：每步完成自动保存到 IndexedDB，刷新不丢失
- **跳过**：Step 3 可跳过，Step 1/2 不可跳过
- **示例体验**：首页提供"查看示例"入口，预填示例数据直接进入 Step 4
- **错误处理**：输入校验失败，输入框下方红色提示，不阻塞其他字段

### 7.2 标签切换交互

- **默认标签**：稳（录取概率 50-80%），最符合"不浪费分数"心智
- **标签内容**：梯度名 + 数量，如"稳 (15)"
- **切换动效**：标签下划线滑动，内容淡入
- **记忆**：切换后记忆选择，刷新保持

### 7.3 志愿卡片设计

#### 卡片结构

| 区域 | 内容 | 交互 |
|------|------|------|
| **头部** | 院校名 · 专业名 + 录取概率徽标 + 梯度标签 | 点击进入详情 |
| **基本信息** | 985/211标签 · 省市 · 选科要求 · 学制 | — |
| **可解释区**（可展开） | "为什么推荐？" 推荐理由列表 | 默认展开首条，其余折叠 |
| **数据来源** | 来源列表（内联引用编号） | 点击跳转来源详情 |
| **操作区** | 加入志愿表 / 对比 / 详情 | 主操作按钮突出 |

#### 录取概率徽标

| 概率范围 | 颜色 | 文案 |
|---------|------|------|
| 80-95% | 蓝色 `#3b82f6` | "保" |
| 50-80% | 绿色 `#16a34a` | "稳" |
| 30-50% | 橙色 `#f59e0b` | "冲" |
| < 30% | 灰色 `#94a3b8` | "难"（默认不展示） |

## 13. 动效设计规范

### 13.1 动效原则

- **快速**：交互反馈 < 100ms
- **自然**：使用 ease-out 缓动
- **有意义**：动效服务于理解，不装饰
- **可关闭**：尊重 `prefers-reduced-motion`

### 13.2 动效规范

| 场景 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 页面切换 | 淡入淡出 | 200ms | ease-out |
| 卡片悬浮 | 上移 2px + 阴影加深 | 150ms | ease-out |
| 按钮点击 | 缩放 0.95 | 100ms | ease-out |
| 标签切换 | 下划线滑动 + 内容淡入 | 200ms | ease-out |
| 列表加载 | 骨架屏 → 内容淡入 | 300ms | ease-out |
| 风险标记 | 色条从左滑入 | 200ms | ease-out |
| 悬浮按钮 | 首次气泡弹出 | 300ms | spring |
| 推荐生成 | 加载动画（绿色脉冲） | 持续 | — |
| 成功提示 | 对勾弹出 | 300ms | spring |

### 13.3 加载状态

- **骨架屏**：列表/卡片加载时显示灰色骨架
- **加载动画**：推荐生成时显示绿色脉冲圆点 + 文案
- **进度提示**：长任务（如测评计算）显示进度条
- **乐观更新**：加入志愿表等操作即时反馈，后台同步

## 14. 无障碍设计

### 15.1 空状态

| 场景 | 文案 | 插图 | 操作 |
|------|------|------|------|
| 推荐结果为空 | "没有找到符合条件的志愿，试试调整筛选条件" | 📭 | "清空筛选" |
| 志愿表为空 | "还没有添加志愿，去推荐结果看看吧" | 📋 | "去推荐" |
| 测评未做 | "还没有测评记录，做个测评了解自己的兴趣吧" | 🧭 | "开始测评" |
| 搜索无结果 | "没有找到相关结果，换个关键词试试" | 🔍 | — |
| AI 未开启 | "AI 问答未开启，前往设置配置 LLM 即可使用" | 💬 | "去设置" |

### 15.2 错误状态

| 场景 | 文案 | 操作 |
|------|------|------|
| 网络错误 | "网络连接失败，请检查网络后重试" | "重试" |
| LLM 调用失败 | "AI 暂时无法响应，请稍后重试" | "重试" / "去设置" |
| 数据加载失败 | "数据加载失败，请刷新页面" | "刷新" |
| 输入校验失败 | 字段下方红色提示 | 自动聚焦 |
| 提交失败 | "提交失败：[原因]"，高风险时拦截 | "修改" |

### 15.3 空状态插图风格

- 线性插画，主色描边
- 简洁，不抢内容焦点
- 尺寸 120×120px
- 配合文案 + 操作按钮

## 16. UX 度量指标

### 16.1 行为指标（埋点采集）

| 指标 | 目标 | 采集方式 |
|------|------|---------|
| 核心路径完成率 | > 85% | 向导 Step1 → 推荐结果 → 加入志愿表 |
| 平均向导耗时 | < 90 秒 | Step1 开始 → Step4 展示 |
| 推荐结果点击率 | > 60% | 卡片详情点击 / 卡片展示 |
| 志愿表添加率 | > 50% | 至少添加 1 个志愿的用户占比 |
| 风险修复率 | > 70% | 高风险被修复 / 高风险总数 |
| AI 使用率 | > 30% | 使用过 AI 的用户占比（LLM 已接入） |
| 溯源点击率 | > 15% | 点击引用编号的用户占比 |

### 16.2 体验指标（用户调研）

| 指标 | 目标 | 采集方式 |
|------|------|---------|
| 推荐结果理解度 | > 80% | 用户调研"是否理解为什么推荐" |
| 信任度 | > 4.0/5 | 评分"是否信任推荐结果" |
| 易用性评分 | > 4.0/5 | SUS 量表 |
| 视觉满意度 | > 4.0/5 | 评分"视觉设计是否舒适" |
| 移动端满意度 | > 3.8/5 | 移动端用户评分 |

### 16.3 性能体验指标

| 指标 | 目标 |
|------|------|
| 首屏加载（FCP） | < 2 秒 |
| 推荐生成耗时 | < 3 秒 |
| 页面切换耗时 | < 200ms |
| 交互响应耗时 | < 100ms |

## 17. 附录

### 17.1 设计决策汇总

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 整体取向 | 极简向导优先 | 3 步完成核心路径，降低使用门槛 |
| 推荐结果信息架构 | 标签切换 | 信息密度高，移动端友好 |
| 导航架构 | 自适应混合 | 桌面顶部 + 移动底部，两端皆优 |
| 视觉风格 | 温暖亲和绿 | 降低焦虑，传递希望与成长 |
| 风险预警交互 | 实时内联标记 + 侧边汇总 | 实时反馈，不打断流程 |
| AI 对话定位 | 辅助频道 + 悬浮助手融合 | 深度咨询 + 快速提问，降级优雅 |
| 数据溯源 | 内联引用标注 | 学术权威，溯源精确 |
